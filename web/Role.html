          # vue权限实现总结

          其实权限这一块是后台管理比较常见的，前端自己可以处理，但是就是有点不太安全，所以还是配合后端一起去实现

          • 使用全局路由守卫实现
          • 实现：前端定义好路由，并且在路由上标记相应的权限信息

          • 全局路由守卫每次都判断用户是否已经登录，没有登录就会调到登录页，已经登录（已经取得了后台返回的用户的权限信息- (角色之类的),判断当前要跳转的路由。是否有权限访问(根据路由权限)之前在role里面设置的根据定义好的界面返回没有权限的*匹配一个404或者没有权限访问

          • 这种方式，菜单可以直接用路由生成(用户没有权限的菜单也会显示),点击跳转的时候才做权限判断，也可以在用户登录后根据用户权限把路由过滤一遍生成菜单(菜单需要保存在vuex利)

            现在基本好多用的是这种方式

            缺点

            • 加载所有的路由，如果路由很多，而用户并不是所有路由都有权限访问，对性能会有影响

          • 全局路由守卫里，每次路由跳转都要做权限判断

          • 菜单信息写死在前端，要改个显示文字或者权限信息，需要重新编译

          • 菜单跟路由耦合在一起，定义路由的 时候还有添加菜单显示标题，突变之类的信息，而且路由不一定作为菜单显示，还要加字端进行标识

          # 登录页与主应用分离

          • 针对前一种实现方式的缺点，可以将登录页与主应用放到不同的页面(不在同一个vue实例里)

          # 实现

          • 登录成功后，进行页面跳转（真正的页面跳转，不是路由的跳转).并将用户权限传递到主应用所在的页面，主应用初始化之前，根据用户权限筛选后的路由作为vue的实例化参数，而不是像前一种方式所有的路由都传递进去，也不需要在全局路由守卫 里做权限判断了

          # 缺点

          • 需要做页面跳转，不是纯粹的单页应用
          • 菜单信息写死在前端,要改个显示文字或权限信息，需要重新编译
          • 菜单跟路由耦合在一起，定义路由的时候还有添加菜单显示标题，图标之类的信息，而且路由不一定作为菜单显示，还需要加字段进行标识

          # 使用addRoutes动态挂在路由

          addRoutes允许在应用初始化之后，动态的挂在路由，有了这个新姿势，就不用像之前那种方式要在应用初始化对路由进行筛选

          # 实现

          • 应用初始化的时候先挂在不需要权限的路由，比如常见的登录页，错误页面404等
          • 有个问题了，addRoutes应该合适调用，在哪里调用
          • 登录后，获取用户的权限信息，然后筛选有权限访问的路由，在调用addRoutes添加路由，这个方法是可以的，但是不能每次进入应用都性需要登录，用户刷新浏览器又要重新登录一次

          所以addRoutes还是要在全路由守卫里进行调用

           



























































             import router from './router'
   import store  from './store'
   import {Message} from 'element-ui'
   import NProgress from 'nprogress'
   import 'nprogress/nprogress.css'
   import {getToken} from '@/utils/auth'

   NProgress.configure({showSpinner:false})
    
   function  hasPermission(roles,permissionRoles){
       if(roles.indexOf('admin'>=0) return true
       if(!permissionRoles) return true 
       return  role.some(role=>permissionRoles.indexOf(role)>=0)
   }

   const whiteList =['/login','/authredirect']

   NProgress.start(){
       if(getToken()){
           if(to.path==='/login'){
               next({path:'/'})
               NProgress.done()
           }else{
           if(stote.getters.roles.length===0){ //判断用户是否拉圈完user_info信息
               store.dispatch('FetUserInfo').then(res=>{//拉取user_info
                   const roles=res.data.roles
                   stroe.dispatch('GenerateRoutes',{roles}).then(()=>{ //根据roles权限生成可以访问的路由表
                   router.addRoutes(store.getter.addRouters)//动态添加可访问路由表
                   next({..to,replace:true}) //hack方法， 确保addRoutes已完成
                   })
               }).catch((err)=>{
                   store.dispatch('FedLogOut').then(()=>{
                       Message.error(err||'Verifictation  failed, please login again')
                       next({path:'/'})
                   })
               })
           }else{
               //没有动态改变权限的需求直接next()删除下方权限判断
               if(hasPermission(store.getter.roles,to.meta.roles)){
                   next()
               }else{
                   next({path:'401',replace:true,query:{noGoBack:true}})
               }
               //可删
           }
       }
       }else{
           if(whiteList.indexOf(to.path)!==-1){ //在免登录白名单没直接进入
             next()
           }else{
               next('/login')//否则全不重定向到登录页
               NProgress.done()
           }
       }
   })

   router.afterEach(()=>{
       NProgress.done()
   })
