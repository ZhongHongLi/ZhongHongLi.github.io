        箭头函数的this默认指向的就是外部的独立作用域

         






















































           let controller={
	   makeRequest:function(){
		   setTimeout(function(){
			   console.log(this) ; // undefined  定时器的this指向的是window
		   })
	   },
	   a:1
   }

 controller.makeRequest();
 
 
 ----
 Es5的写法
 let controller={
 	   makeRequest:function(){
		   var that=this;  //在这一块定义一个当前作用域的变量that接收this
 		   setTimeout(function(){
 			   console.log(that.a) ; =>  a 
 		   })
 	   },
 	   a:1
   }
 
 controller.makeRequest();

  let controller={

  controller.makeRequest();

        # Promise (难点)

        Promise作为ES6中推出的新的概念，改变了JS的异步编程，现代前端大部分的异步请求都是使用Promise实现，fetch这个web api也是基于Promise的，这里不得简述一下之前统治JS异步编程的回调函数，回调函数有什么缺点，Promise又是怎么改善这些缺点

        回调函数,js是单线程的,多线程改变DOM的话导致页面紊乱，且回调导致代码难以维护导致回调地狱

         









            ajax('http://localhost:3000',()=>{
		 ajax('http://localhost:3001',()=>{
			ajax('http://localhost:3002',()=>{
				ajax('http://localhost:3003',()=>{
							 
				})		 
			}) 
	    })
	})

        缺点: 1.多重嵌套,导致回调地狱

        2.代码跳跃,并非人类习惯的思维模式

        3.信任问题，你不能把你的回调完全寄托给第三方库,因为你不知道第三方库到底会怎么执行回调（多次执行）

        4.第三库可能没有提供错误处理

        5.不清楚回调是否都是异步调用的（可以同步调用ajax，在收到响应前会阻塞整个线程，会陷入假死状态，非常不推荐）

        xhr.open('GET','01.json',fasle); //第三个async设为false 就是同步可以调用ajax

        接下来就要将Promise了,Promise是一个构造函数,通过new关键字创建一个Promise的实例

         











           //看看Promise如何解决回调函数这些问题
   
   let promise =new Pronmise((resolve,reject)=>{
  	 setTimeout(()=>{
  		 resolve('i have is resolved');
  	 },2000)
   })
  promise.then(res=>{
  	console.log(res) //2秒后打印
